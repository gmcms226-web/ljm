## 📖 프로젝트 소개

이 프로젝트는 **이정민 애견미용학원**의 학원 소개, 교육과정, 공지사항, 학생 실습 갤러리, 오시는 길, 상담 신청 정보를 제공하기 위해 제작한 웹사이트입니다.  
HTML, CSS, Vanilla JavaScript를 기반으로 제작했으며, Swiper 슬라이더, Firebase Auth, 네이버 지도 API, 현재 위치 기반 거리 계산 기능을 적용했습니다.  
페이지별 CSS를 분리하고 공통 헤더·푸터·모달·TOP 버튼을 반복 적용하여 실제 학원 홈페이지처럼 사용할 수 있도록 구성했습니다.

### 네이버 지도 + 현재 위치 거리 계산

```javascript
// Haversine 공식으로 사용자 위치 → 학원 거리 계산
function getDistance(lat1, lng1, lat2, lng2) {
  const R = 6371;
  const dLat = (lat2 - lat1) * Math.PI / 180;
  const dLng = (lng2 - lng1) * Math.PI / 180;
  const a = Math.sin(dLat/2) ** 2 +
            Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
            Math.sin(dLng/2) ** 2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
}
```

// IntersectionObserver로 스크롤 시 갤러리 아이템 순차 등장

## 🐛 Trouble Shooting

<details> <summary><b>네이버 지도 API 인증 실패 문제</b></summary>

**문제**
네이버 지도가 잠시 표시된 뒤 “Open API 인증이 실패했습니다.” 메시지가 출력됨.

**원인**
네이버 클라우드 플랫폼의 Web 서비스 URL 등록과 실제 Live Server 접속 주소가 일치하지 않거나, 설정 반영 시간이 지연됨.

**해결**
http://127.0.0.1:5500, http://localhost:5500을 모두 등록하고, 저장 후 일정 시간 뒤 새로고침하여 해결.

</details> <details> <summary><b>갤러리 이미지 레이아웃 깨짐 문제</b></summary>

**문제**
학생실습 이미지가 2열 그리드가 아닌 한 장씩 크게 표시됨.

**원인**
이미지 카드들을 감싸는 .practice-photo-grid 요소가 누락되어 grid CSS가 적용되지 않음.

**해결**
8개의 이미지 카드를 .practice-photo-grid 내부에 배치하여 정상적인 2열 레이아웃으로 수정.

</details> <details> <summary><b>로그인 모달이 일부 페이지에서 작동하지 않는 문제</b></summary>

<details>
<summary><b>로그인 상태에서 헤더 버튼 클릭 시 모달이 함께 열리는 문제</b></summary>

**문제**  
로그인 상태에서 헤더 버튼 클릭 시 로그아웃 처리와 동시에 `common.js`의 로그인 모달 오픈 리스너가 중복 실행됨.

**원인**  
`common.js`와 `firebase-auth.js` 두 파일이 동일한 버튼에 이벤트 리스너를 등록하여 이벤트 버블링으로 중복 실행.

**해결**  
로그아웃 핸들러에 캡처 페이즈(`capture: true`) + `stopImmediatePropagation()` 적용.

```javascript
loginBtn.addEventListener('click', handleLogout, true);
function handleLogout(e) {
  e.stopImmediatePropagation();
  signOut(auth);
}
```

</details>

## 💬 프로젝트 회고

이번 프로젝트는 실제 운영 중인 웹사이트를 참고하며, 사용자 입장에서 불편하게 느껴지는 정보 구조와 화면 흐름을 분석하는 것에서 시작했습니다. 단순히 화면을 따라 만드는 것이 아니라, 학원 소개, 교육과정, 공지사항, 학생 실습, 오시는 길, 상담 신청처럼 사용자가 실제로 필요로 하는 핵심 정보를 중심으로 페이지를 재구성했습니다.

기획 단계에서는 필요한 메뉴와 페이지 구조를 먼저 정리하고, 이후 디자인 시안 제작, HTML/CSS 구현, 반응형 수정, Firebase Auth와 네이버 지도 API 연동까지 단계적으로 진행했습니다. 작업 과정에서 예상하지 못한 오류도 많았지만, ChatGPT, Claude, Gemini 등 여러 AI 도구를 활용해 문제 원인을 찾고 해결 방법을 비교하며 직접 수정해나갔습니다. 그 과정을 통해 단순히 코드를 복사하는 것이 아니라, 구조를 이해하고 적용하는 경험을 할 수 있었습니다.

아쉬웠던 점은 프로젝트 초반에 컬러 시스템과 폰트 사용 규칙을 더 명확하게 정하지 못한 부분입니다. 페이지가 늘어나면서 색상과 여백, 폰트 크기를 후반부에 다시 맞춰야 하는 상황이 생겼고, 디자인 시스템을 초기에 정리하는 것이 유지보수에 얼마나 중요한지 체감했습니다. 다음 프로젝트에서는 기획 단계에서 컬러 팔레트, 폰트 규칙, 버튼 스타일, 섹션 여백 등을 먼저 정의한 뒤 작업을 시작해 더 일관성 있는 결과물을 만들고 싶습니다.
